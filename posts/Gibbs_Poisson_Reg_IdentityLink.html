An Efficient Gibbs Sampler for Poisson Regression with Identity Link¶

A Poisson regression model with identity link is given as follows:

$y_i\sim\mbox{Pois}\bigg(\sum_{j=1}^{p}x_{ij}\lambda_j\bigg),$

where $y_i\in\mathcal{N}_{+}$ is the observed count, $x_{ij}\in\Re_{+}$'s are the predictors assocaited with the observation and $\lambda_j\in\Re_{+}$'s are the regression coefficients. Note that for the model to work, one only needs $\sum_{j=1}^{p}x_{ij}\lambda_j$ to be positive. However, for simplicity, both $x_{ij}$'s and $\lambda_{j}$ are constrained to be positive.

By introducing auxiliary variables $\eta_{ij}\in\mathcal{N}_{+}$, the model can be equivalently written as

$y_i=\sum_{j=1}^{p}\eta_{ij},$

$\eta_{ij}\sim\mbox{Pois}\bigg(x_{ij}\lambda_j\bigg)$ for $j=1,\ldots,p$,

Suppose gamma priors are put over $\lambda_j$'s:

$\lambda_j\sim\mbox{Gamma}(a,b),$

then with little effort one can derive the full conditional for $\lambda_j$ conditional on all the $\eta_{ij}$'s:

$\eta_{ij}|-\sim\mbox{Gamma}(\sum_{i=1}^{n}\eta_{ij}+a,\sum_{i=1}^{n}x_{ij}+b).$

To complete the Gibbs sampler, I will next give the full conditionals for $\{\eta_{ij}\}_{j=1}^{p}$'s conditonal on $y_i$, $x_{ij}$ and $\{\lambda_{j}\}_{j=1}^{p}$. Directly using the result from Relationship Between Poisson and Multinomial Distributions, the full conditional is

$\eta_{i1},\ldots,\eta_{ip}|y_i,x_{ij}\lambda_{1},\ldots,x_{ij}\lambda_{p} \sim \mbox{Multinomia}\bigg(y_i,\dfrac{x_{i1}\lambda_1}{\sum_{j=1}^{p}x_{ij}\lambda_j},\ldots,\dfrac{x_{ip}\lambda_p}{\sum_{j=1}^{p}x_{ij}\lambda_j}\bigg)$.

A pretty nice thing about the above updating rule is that when $y_i$'s are sparse, i.e., most of $y_i$'s are zeros, one does not need to update the corresponding $\eta_{ij}$'s since they are immediately all zeros. This could save a huge amount of computation time.

The Gibbs Sampler¶

For all $i$ such that $y_i==0$, fix $\eta_{ij}=0$ for all $j$, then do the following iteration:

  • Sample for all $i$ such that $y_i>0$, sample $\eta_{i1},\ldots,\eta_{ip}$ from $\mbox{Multinomia}\bigg(y_i,\dfrac{\lambda_1}{\sum_{j=1}^{p}\lambda_j},\ldots,\dfrac{\lambda_p}{\sum_{j=1}^{p}\lambda_j}\bigg)$;
  • For all $j$, sample $\lambda_j$ from $\mbox{Gamma}(\sum_{i=1}^{n}\eta_{ij}+a,\sum_{i=1}^{n}x_{ij}+b)$.

import numpy as np

def poissregGibbs(y,x,nb,ns):
    """ Gibbs sampler for Poisson regression with identity link
        
        Args:
            y: np.array, responses
            x: np.array, predictors
            nb: int, number of burn-ins
            ns: int, number of after-burnin samples
    """
    # pre-processing
    n,p = x.shape
    nzId = [i for i in range(n) if y[i]] # indices for non-zero y's
    nNz = len(nzId) # total number of non-zero y's
    nZ = n-nNz # total number of zero y's
    Id2Nid = {nzId[i]:i for i in range(nNz)}
    # initial values for lmbda
    lmbda = np.array( [1.0/p for __ in range(p)] )
    # intermidiate values for eta's
    eta = np.array( [[0 for __ in range(p)] for __ in range(nNz)] )
    # store sampled values
    ret = np.array( [[0.0 for __ in range(ns)] for __ in range(p)] )
    for it in range(nb+ns):
        if not (it+1) % max(1,(nb+ns)/10):
            print str(it+1)+" iterations completed"
        # update eta's
        for i in nzId:
            tmp = np.multiply(x[i,:],lmbda)
            probs = tmp/np.sum(tmp)
            eta[ Id2Nid[i],: ] = np.random.multinomial(y[i], probs)
        # update lmbda
        for j in range(p):
            lmbda[j] = np.random.gamma(shape=2.0+np.sum(eta[:,j]), scale=1.0/(2.0+np.sum(x[:,j])))
        if it >= nb:
            ret[:,it-nb] = lmbda
    return ret

The sampler ran for 40 seconds. Suppose one wants to collect 15000 samples then the sampler should roughly run 10 minutes based on simple multiplying 40 seconds with 15. Now let us take a closer look at the computational cost of the sampler for each iteratation.

Note that in this simulated dataset 90% of the $y$'s are zero. But in reality one may observe a much more severe sparsity. The computational cost of the proposed Gibbs sampler here scales linearly in the number of non-zero $y$'s so in those cases the Gibbs sampler could be even faster.
